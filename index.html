    • Porsche GT3 RS

    El Porsche 911 GT3 RS es la versión más radical y orientada a circuito de la gama 911. Representa la máxima expresión de ingeniería de alto rendimiento de Porsche, diseñado para ofrecer una experiencia de conducción cercana a la de un vehículo de competición pero homologado para carretera.

    Características principales

    El GT3 RS cuenta con un motor bóxer atmosférico de 6 cilindros y 4.0 litros que produce aproximadamente 525 CV (según el modelo y año). Alcanza una velocidad máxima de más de 290 km/h y puede acelerar de 0 a 100 km/h en aproximadamente 3.2 segundos.

    Tecnología de competición

    Incorpora numerosas tecnologías derivadas directamente de la competición, como suspensión activa, aerodinámica avanzada con un gran alerón trasero ajustable, y materiales ligeros como fibra de carbono para reducir el peso. El sistema de escape deportivo proporciona un sonido característico que refleja su naturaleza de alto rendimiento.

    Diseño y aerodinámica

    Su diseño está completamente orientado a la funcionalidad aerodinámica. Cada elemento, desde las tomas de aire hasta el difusor trasero, está diseñado para optimizar el flujo de aire y generar carga aerodinámica. El gran alerón trasero es uno de sus elementos más distintivos y funcionales.

    El Porsche GT3 RS representa la perfecta combinación entre un coche de carreras y un vehículo homologado para carretera, ofreciendo sensaciones de conducción puras y directas.

    15 Tips para HTML

    Estos son 15 consejos básicos para mejorar tu código HTML y crear mejores sitios web:

    1. Usa alt descriptivo en imágenes

      Consejo: Mejora la accesibilidad para lectores de pantalla.

      <img src="cetis125.jpg" alt="Logotipo de CETis 125">
    2. Usa header, main, footer correctamente

      Consejo: Mejora la semántica de tu documento HTML.

    3. Agrega un favicon

      Consejo: Personaliza tu pestaña del navegador.

      <link rel="icon" href="favicon.ico" type="image/x-icon">
    4. Usa variables CSS más completas

      Consejo: Añade colores como --color-border y --color-dark que usas, pero no defines.

      :root {
      --color-secondary: #166088;
      --color-gray: #666;
      --color-dark: #222;
      --color-border: #ddd;
      }
    5. Agrega un hover al logo para un toque animado

      Consejo: Añade un efecto que al pasar el cursor sobre el objeto se agrande. Puedes cambiarle la forma, color entre otras.

      .logo img:hover {
      transform: scale(1.1);
      }
    6. Añade aria-label a enlaces si el texto no es suficientemente claro

      Consejo: Mejora la accesibilidad en los botones de navegación.

      <a href="#" class="nav-link" aria-label="Enlace a mi presentación">Mi Presentación</a>
    7. Usa <section> con aria-labelledby para accesibilidad
      <section aria-labelledby="titulo-html">
      <h2 id="titulo-html" class="section-title">Historia del HTML</h2>
      </section>
    8. Añade un espacio entre todos los objetos

      Consejo: para añadir un espacio entre los objetos como divs o enlaces, puedes añadir un gap.

      css {
      gap: 30px;
      }
    9. Evita enlaces que abren pestañas sin advertir

      Consejo: Usa rel="noopener noreferrer" para seguridad.

      <a href="..." target="_blank" rel="noopener noreferrer">Mi blogger</a>
    10. Añade transiciones suaves a secciones

      Consejo: Agrega animación al hacer scroll.

      section {
      transition: opacity 0.5s ease;
      }
    11. Mejora legibilidad con mayor contraste

      Consejo: Usa herramientas como WebAIM para revisar el contraste de color texto/fondo.

      color: #000; /* Mejor que #666 para fondo claro */
    12. Usar Google Fonts en html

      Consejo: Lo puedes usar para cambiar la fuente de un texto y que no se vea básica.

      <link href="https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;500;600&display=swap" rel="stylesheet">
    13. Añade scroll suave

      Consejo: quita la decoración de un texto, o sea la línea del texto.

      css {
      text-decoration: none;
      }
    14. Cambia la forma de un cursor cuando lo pasas sobre un objeto

      Consejo: cuando le des la clase a un div, img o cualquier objeto puedes cambiarle el cursor utilizando:

      css {
      cursor: pointer; /* este cursor cambia la forma al cursor selectivo */
      }
    15. Añade objetos o nav fijados

      Consejo: Para hacer que un nav o objeto esté fijado al deslizar el cursor para abajo utiliza:

      nav {
      position: sticky;
      }

    Historia del HTML

    El HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Su historia está estrechamente ligada al nacimiento de la World Wide Web.

    Todo comenzó en los laboratorios del CERN (Organización Europea para la Investigación Nuclear) en Ginebra, Suiza. En 1989, Tim Berners-Lee, un científico británico, propuso un sistema de hipertexto para compartir documentos entre investigadores. Este sistema sería la base de lo que hoy conocemos como la World Wide Web.

    Puntos clave en la evolución del HTML

    El nacimiento del HTML (1991)

    Tim Berners-Lee creó el HTML como un lenguaje simple para estructurar y presentar contenido en la web. La primera versión contenía solo 18 elementos, incluyendo encabezados, párrafos y enlaces básicos. No existían elementos para imágenes, tablas o formularios. El objetivo principal era permitir a los científicos compartir documentos de investigación a través de Internet.

    Estandarización y crecimiento (1993-1997)

    Con la creación del W3C (World Wide Web Consortium) en 1994, HTML comenzó a estandarizarse. HTML 2.0 se publicó en 1995 como el primer estándar oficial, añadiendo formularios y soporte para gráficos. HTML 3.2 (1997) introdujo tablas, applets y opciones de texto fluido alrededor de imágenes, expandiendo significativamente las capacidades de diseño web.

    La era de los navegadores y la guerra de navegadores (1995-2000)

    Este período vio el surgimiento de Netscape Navigator e Internet Explorer, que comenzaron a implementar sus propias extensiones de HTML. Esto llevó a la "guerra de navegadores", donde los sitios web a menudo tenían que ser diseñados dos veces para funcionar correctamente en diferentes navegadores. Esta fragmentación impulsó la necesidad de estándares más sólidos.

    HTML 4 y la separación de contenido y presentación (1997-1999)

    HTML 4.0 marcó un cambio filosófico importante: la separación del contenido (HTML) de la presentación (CSS). Se desaconsejaron los elementos puramente visuales como <font> y <center>, favoreciendo el uso de hojas de estilo en cascada (CSS) para el diseño. Esta versión también mejoró la accesibilidad y el soporte para scripts.

    El experimento XHTML (2000-2008)

    XHTML fue un intento de reformular HTML como una aplicación XML, con reglas más estrictas como etiquetas cerradas obligatoriamente y uso de minúsculas. Aunque técnicamente superior en algunos aspectos, XHTML 2.0 fue demasiado estricto y no compatible con versiones anteriores, lo que limitó su adopción. Este enfoque eventualmente fue abandonado en favor de HTML5.

    La revolución HTML5 (2008-2014)

    HTML5 representó un cambio radical, enfocándose en aplicaciones web prácticas en lugar de pureza teórica. Introdujo elementos semánticos como <article>, <section>, <nav> y <header>, que describen el propósito del contenido. También añadió soporte nativo para audio y video, canvas para gráficos, almacenamiento local y aplicaciones web offline. HTML5 fue diseñado para ser compatible con versiones anteriores, permitiendo una transición más suave.

    HTML Living Standard (2016-Presente)

    En lugar de versiones numeradas, HTML ahora evoluciona como un "estándar vivo" que se actualiza continuamente. El WHATWG (Web Hypertext Application Technology Working Group) mantiene esta especificación, permitiendo que el lenguaje se adapte rápidamente a las necesidades cambiantes de la web. Las mejoras recientes incluyen mejor soporte para dispositivos móviles, nuevas APIs para realidad virtual y aumentada, y funcionalidades para aplicaciones web progresivas.

    La evolución del HTML refleja cómo ha cambiado la web a lo largo de los años. Lo que comenzó como un simple lenguaje para compartir documentos científicos se ha convertido en la base de complejas aplicaciones web, sitios interactivos y experiencias multimedia.

    Hoy en día, HTML5 junto con CSS3 y JavaScript forman el trío fundamental de tecnologías para el desarrollo web. El HTML moderno se centra en la semántica, accesibilidad y compatibilidad entre dispositivos, permitiendo experiencias web ricas en cualquier pantalla, desde teléfonos móviles hasta televisores inteligentes.

    Impacto del HTML

    El HTML ha tenido un impacto incalculable en nuestra sociedad. Ha democratizado la publicación de información, permitiendo que cualquier persona con acceso a Internet pueda crear y compartir contenido.

    Democratización de la información

    Antes de HTML, publicar información requería recursos significativos (imprentas, editoriales, etc.). HTML permitió que cualquier persona con acceso a un ordenador pudiera crear y compartir contenido globalmente, cambiando fundamentalmente cómo se distribuye el conocimiento.

    Transformación de industrias

    El comercio electrónico, la educación en línea, el entretenimiento digital y prácticamente todos los sectores han sido transformados por la web basada en HTML. Industrias enteras han surgido o se han reinventado gracias a esta tecnología.

    Conectividad global

    HTML ha ayudado a conectar personas de todo el mundo, permitiendo la comunicación instantánea y el intercambio cultural a una escala sin precedentes. Las redes sociales, los blogs y los foros, todos construidos con HTML, han creado comunidades globales.

    Sin HTML, Internet tal como lo conocemos no existiría, y nuestra forma de comunicarnos, trabajar y entretenernos sería completamente diferente. Es uno de los inventos más influyentes de la era moderna, comparable en su impacto a la imprenta o el teléfono.
